# 1. Product Overview

## 1.1 Description

An IoT-based system for remote monitoring and control of irrigation systems, with local equipment protection capabilities.

## 1.2 Product Goal

The goal of the product is to reduce the need for farmers to be physically present at the field in order to monitor or control irrigation operation.

By enabling remote supervision and control, the product helps farmers save man-hours, respond faster when something goes wrong, and gain peace of mind when they are busy with other work or daily life.

The product adds a remote layer of visibility and control while keeping the existing local/manual operation available.

## 1.3 Product Scope

The product covers the remote monitoring and control of irrigation systems through an IoT-based system.

The product targets existing irrigation systems using a START (NO)/STOP (NC) latching cirquit for the control of the main pump.

It includes the ability to observe irrigation operation, collect operational data, send remote Start/Stop commands, verify whether commands were executed, notify users about important events, and support local equipment protection.

The product also supports historical visibility into system operation, including operating hours, pressure behavior, energy consumption, and system faults.

The product keeps the existing local/manual operation available, so the remote layer complements the existing irrigation control method rather than replacing it.

## 1.4 System Functionality

This section describes the main system functionality in plain language for users, installers, partners, and other non-technical stakeholders.

Remote functionality is available from anywhere as long as the user has internet connectivity.

### 1.4.1 Remote Monitoring

Users can view the irrigation system’s operating state remotely.

### 1.4.2 Remote Control

Users can request Start or Stop operation remotely.

### 1.4.3 Field-Based Confirmation

After a remote command is requested, the system uses field data to inform the user whether the requested action was actually performed.

### 1.4.4 Alerts and Notifications

Users are notified when important events occur such as command failure, operating state changes, power state changes, pressure deviations, or automatic shutdowns.

### 1.4.5 Power Loss and Restoration Alerts

Users are notified when electrical power is lost and when it comes back.

### 1.4.6 Manual Operation Compatibility

The existing local/manual way of operating the irrigation system remains available.

### 1.4.7 Flexible Connectivity

The system is intended to support SIM, Wi-Fi, and Ethernet connectivity.

### 1.4.8 Multiple Devices per User

Users can monitor and control multiple irrigation devices through a unified digital environment.

### 1.4.9 Multiple Users per Device

Multiple users can access the same device with distinct roles and permissions.

### 1.4.10 Historical Operation Data

Users can review historical operation information, including operating hours, pressure behavior, energy consumption, and system faults.

### 1.4.11 Local Equipment Protection

The system can protect equipment locally, even without connectivity, by stopping operation in unsafe conditions, such as dry run or pressure limit violations.
